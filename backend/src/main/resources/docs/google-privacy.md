# Sign in with Google & Data Use Policy

*Last updated: July 21, 2026*

## 1. Google Sign-In Integration
${APP_NAME} offers optional Sign in with Google to authenticate users and enable seamless cross-device encrypted state synchronization.

## 2. Data Accessed & Used
When you sign in with Google, we request the following permissions:
- **Email & Profile** (`openid`, `email`, `profile`): Used solely to identify your account session.
- **Google Drive AppData** (`https://www.googleapis.com/auth/drive.appdata`): Used to store client-side encrypted key backup files in your private application data folder.

## 3. Google API Limited Use Disclosure
${APP_NAME}'s use and transfer of information received from Google APIs to any other app will adhere to the [Google API Services User Data Policy](https://developers.google.com/terms/api-services-user-data-policy), including the Limited Use requirements.

## 4. Data Sharing & Security
We do not sell, share, or transfer your Google user data to third parties, advertisers, or external data brokers.
